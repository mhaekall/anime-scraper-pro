🚀 1. Executive Summary

Fokus Ulasan: Evaluasi arsitektur frontend dan backend khusus untuk halaman streaming video (Watch Page) pada repository anime-scraper-pro, beserta rancangan teknis peningkatan fungsionalitas UI/UX media player.

Stack Teknologi Repository:

  • Frontend: Next.js 15, React 19, Tailwind CSS 4, Zustand 5, SWR, hls.js (Dynamic Import).
  • Backend: Python FastAPI, mekanisme Stream Proxy v1, SSRF validation, Resilient 3-tier mapping resolution di stream_v2.py.

Kelebihan Saat Ini

Fondasi teknis proyek ini sudah sangat solid secara backend. Pendekatan proxy streaming langsung mem-bypass kebutuhan iframe lintas origin, dan 3-tier reconciler memastikan ketahanan link yang luar biasa tinggi. Struktur app-store.ts menggunakan Zustand 5 juga sudah disiapkan dengan matang.

Overall Verdict

Meskipun arsitektur under-the-hood sangat mutakhir, antarmuka pengguna pada halaman streaming saat ini masih sangat barebones. Elemen krusial seperti menu pengaturan terstruktur, auto-next episode, navigasi episode di area player, dan kontrol gestur mobile sama sekali belum diimplementasikan. Fitur di store (Zustand) masih berstatus ghost features. Dokumen ini mendefinisikan langkah konkret untuk mengangkat standar player ini menjadi sekelas platform big tech (YouTube, Bstation, Netflix).

🔍 2. Current State Audit — Halaman Streaming

Berdasarkan penelusuran file aktual di repository, berikut adalah status terkini dari komponen-komponen penyusun halaman streaming.

  • Kekurangan UX: Tidak ada tombol Prev/Next di dekat player. Sidebar tidak memiliki informasi anime (sinopsis, genre, dll). Tab komentar masih placeholder. Tidak ada opsi pindah provider/server.

Status: Functional but Disconnected

  • Fakta: Menampilkan list dengan thumbnail, bar progress riwayat tontonan, indikator selesai, dan sistem lazy load per 50 item.
  • Kekurangan UX: Tidak ada highlight atau auto-scroll ke episode yang saat ini sedang diputar. Ketiadaan kolom search/filter menyulitkan navigasi anime berepisode ratusan.

Status: Store Siap, UI Belum Terhubung

  • Fakta: Memiliki deklarasi state autoPlayNext: true, skipIntro: false, dan videoQuality: "Auto".
  • Kekurangan UX: Semua pengaturan ini adalah ghost features. Player saat ini mengabaikan autoPlayNext maupun skipIntro, membuat pengaturan tersebut tidak berguna di UI.

Pengalaman menonton secara beruntun (binge-watching) saat ini terputus total karena tidak adanya navigasi Auto-Next atau tombol episode selanjutnya di area player. Pengguna harus kembali ke sidebar, mencari episode berikutnya secara manual setiap 20 menit.

4.3 Wiring onEnded & 90% Threshold (di VideoPlayer.tsx)

import { useSettings } from "@/core/stores/app-store";

// Props penambahan untuk trigger auto-next
interface VideoPlayerProps {
  // ... props existing ...
  onRequireAutoNext: () => void;
}

export function VideoPlayerInner({ onRequireAutoNext /* ... */ }: VideoPlayerProps) {
  const autoPlayNext = useSettings((s) => s.settings.autoPlayNext);
  const [hasTriggeredAutoNext, setHasTriggeredAutoNext] = useState(false);

  useEffect(() => {
    const video = videoRef.current;
    if (!video) return;

    const handleTimeUpdate = () => {
      const current = video.currentTime;
      const dur = video.duration;
      // Trigger threshold: 90% completion
      if (dur > 0 && (current / dur) >= 0.9 && autoPlayNext && !hasTriggeredAutoNext) {
        onRequireAutoNext();
        setHasTriggeredAutoNext(true);
      }
    };

    const handleEnded = () => {
      if (autoPlayNext && !hasTriggeredAutoNext) {
        onRequireAutoNext();
        setHasTriggeredAutoNext(true);
      }
    };

    video.addEventListener("timeupdate", handleTimeUpdate);
    video.addEventListener("ended", handleEnded);

    return () => {
      video.removeEventListener("timeupdate", handleTimeUpdate);
      video.removeEventListener("ended", handleEnded);
    };
  }, [autoPlayNext, hasTriggeredAutoNext, onRequireAutoNext]);

  // ... rest of component
}

🏃 5. Saran Peningkatan #3: Skip Opening / Intro Button

Lagu opening anime rata-rata berdurasi 90 detik. Keberadaan tombol ini bisa secara masif mengurangi repetisi yang dirasakan user. Cukup tampilkan tombol di window timestamp tertentu (contoh: 00:30 hingga 02:00) yang akan menghilang jika dilewati.

frontend/ui/player/SkipIntroButton.tsxReact / TypeScript
"use client";
import { useState, useEffect } from 'react';
import { useSettings } from '@/core/stores/app-store';

interface SkipIntroProps {
  currentTime: number;
  onSkip: (targetTime: number) => void;
  // Default window umum: lagu mulai di detik 30, selesai di detik 120 (1m 30s durasi)
  showAt?: number; 
  hideAt?: number;
}

export function SkipIntroButton({ currentTime, onSkip, showAt = 30, hideAt = 120 }: SkipIntroProps) {
  const autoSkip = useSettings(s => s.settings.skipIntro);
  const [hasSkipped, setHasSkipped] = useState(false);

  // Munculkan hanya di dalam window time, dan jika belum ditekan
  const isVisible = currentTime >= showAt && currentTime <= hideAt && !hasSkipped;

  // Fitur "Ghost": Auto-Skip tanpa klik jika diaktifkan di store
  useEffect(() => {
    if (autoSkip && isVisible) {
      onSkip(hideAt);
      setHasSkipped(true);
    }
  }, [autoSkip, isVisible, hideAt, onSkip]);

  if (!isVisible) return null;

  return (
    <button 
      onClick={(e) => { 
        e.stopPropagation(); 
        onSkip(hideAt); 
        setHasSkipped(true); 
      }}
      className="absolute bottom-24 right-6 z-30 px-5 py-2.5 bg-black/80 hover:bg-black text-white text-sm font-bold rounded-lg border border-white/20 shadow-2xl backdrop-blur-md flex items-center gap-2 animate-in slide-in-from-right-4 duration-300"
    >
      Lewati Opening <span className="text-[#0a84ff]">▶▶</span>
    </button>
  );
}

6.1. Mobile Gesture Controls (Hook)

frontend/core/hooks/useVideoGestures.tsReact / TypeScript
import { useState, useRef, useEffect, MutableRefObject } from 'react';

/**
 * Hook untuk mengenali gestur double-tap kiri/kanan pada kontainer video
 */
export function useVideoGestures(videoRef: MutableRefObject<HTMLVideoElement | null>) {
  const containerRef = useRef<HTMLDivElement>(null);
  const [ripple, setRipple] = useState<{ side: 'left'|'right', id: number } | null>(null);
  const lastTapRef = useRef<number>(0);

  const handleTouchStart = (e: React.TouchEvent | React.MouseEvent) => {
    if (!videoRef.current || !containerRef.current) return;
    
    const now = Date.now();
    const DOUBLE_TAP_DELAY = 300;
    
    // Deteksi Double Tap
    if (now - lastTapRef.current < DOUBLE_TAP_DELAY) {
      e.preventDefault(); // Mencegah zoom bawaan browser
      
      let clientX;
      if ('touches' in e) clientX = e.touches[0].clientX;
      else clientX = (e as React.MouseEvent).clientX;

      const rect = containerRef.current.getBoundingClientRect();
      const isRightSide = (clientX - rect.left) > (rect.width / 2);
      
      // Hitung lompatan waktu (10 detik)
      const skipAmount = isRightSide ? 10 : -10;
      videoRef.current.currentTime = Math.max(0, Math.min(videoRef.current.duration, videoRef.current.currentTime + skipAmount));
      
      // Trigger animasi ripple
      setRipple({ side: isRightSide ? 'right' : 'left', id: now });
      setTimeout(() => setRipple(null), 500); // Bersihkan ripple
      
      lastTapRef.current = 0; // Reset
    } else {
      lastTapRef.current = now;
    }
  };

  return { containerRef, handleTouchStart, ripple };
}

  return (
    <div className="min-h-screen bg-[#0a0c10] flex flex-col pt-0 lg:pt-14">
      
      {/* Layout Grid: berubah dinamis mengikuti Theater Mode */}
      <div className={`w-full max-w-[1600px] mx-auto flex flex-col ${isTheater ? 'lg:flex-col' : 'lg:flex-row'} gap-6 px-0 lg:px-6`}>
        
        {/* SEKSI KIRI: Player & Info */}
        <div className={`flex-1 flex flex-col w-full min-w-0 transition-all duration-300`}>
          
          {/* Player Container dengan rasio aspek terjaga */}
          <div className="relative w-full aspect-video bg-black lg:rounded-2xl overflow-hidden shadow-2xl border border-white/5 sticky top-0 lg:static z-20 group">
            <VideoPlayer 
              sources={sources.filter(s => s.provider === activeServer)} 
              animeSlug={anime.id} 
              episodeNum={currentEpisode.number} 
              onRequireAutoNext={() => setShowAutoNext(true)}
            />

            {/* Render AutoNextOverlay bila ditrigger */}
            {showAutoNext && nextEp && (
              <AutoNextOverlay 
                nextEpisodeUrl={nextEp.url} 
                nextEpisodeTitle={`Episode ${nextEp.number}`}
                nextThumbnail={nextEp.thumbnailUrl}
                isLastEpisode={false}
                onCancel={() => setShowAutoNext(false)}
              />
            )}
            {showAutoNext && !nextEp && (
              <AutoNextOverlay isLastEpisode={true} onCancel={() => setShowAutoNext(false)} nextEpisodeUrl="" nextEpisodeTitle="" />
            )}
          </div>

          {/* Navigation Bar di bawah player */}
          <EpisodeNavigationBar prevUrl={prevEp?.url} nextUrl={nextEp?.url} currentTitle={`${anime.title} - Ep ${currentEpisode.number}`} />

          {/* Action Bar & Server Selection */}
          <div className="flex flex-wrap items-center justify-between gap-4 py-4 px-4 lg:px-0">
            <div className="flex gap-2 items-center">
              <span className="text-xs text-[#8e8e93] font-bold uppercase tracking-wider">Server:</span>
              {['oploverz', 'otakudesu', 'samehadaku'].map(server => (
                <button 
                  key={server} onClick={() => setActiveServer(server)}
                  className={`px-3 py-1.5 rounded-md text-xs font-bold transition-colors ${activeServer === server ? 'bg-[#0a84ff] text-white' : 'bg-[#1c1c1e] text-[#8e8e93] border border-white/10 hover:text-white'}`}
                >
                  {server.toUpperCase()}
                </button>
              ))}
            </div>
            <div className="flex gap-3">
              <button onClick={() => setIsTheater(!isTheater)} className="hidden lg:flex items-center gap-2 px-4 py-2 bg-[#1c1c1e] hover:bg-[#2c2c2e] rounded-full text-sm font-bold text-white border border-white/10">
                {isTheater ? 'Default Mode' : 'Theater Mode'}
              </button>
            </div>
          </div>

          {/* Anime Meta Info Ala YouTube */}
          <div className="px-4 lg:px-0 py-2 mb-8">
            <h1 className="text-2xl font-black text-white mb-2">{anime.title} - Episode {currentEpisode.number}</h1>
            
            <div className="bg-[#1c1c1e]/60 p-4 rounded-xl border border-white/5">
              <div className="flex flex-wrap items-center gap-4 mb-3 text-sm">
                <span className="text-white font-bold">⭐ {anime.score} / 100</span>
                <span className="text-[#8e8e93]">Rilis: {currentEpisode.airDate || 'N/A'}</span>
                <span className="text-[#8e8e93]">Studio: {anime.studios.join(', ')}</span>
              </div>
              <div className="flex gap-2 mb-3">
                {anime.genres.map(g => <span key={g} className="text-xs bg-[#0a84ff]/20 text-[#64d2ff] px-2 py-1 rounded">{g}</span>)}
              </div>
              <p className="text-sm text-[#e5e5ea] leading-relaxed line-clamp-3 hover:line-clamp-none cursor-pointer">
                {anime.synopsis}
              </p>
            </div>
          </div>

        </div>

        {/* SEKSI KANAN: Sidebar Daftar Episode */}
        <div className={`w-full ${isTheater ? 'lg:w-full' : 'lg:w-[380px] shrink-0'} flex flex-col`}>
          <div className="bg-[#1c1c1e] rounded-xl border border-white/5 overflow-hidden flex flex-col max-h-[800px]">
            
            {/* Tabs */}
            <div className="flex border-b border-white/10">
              <button onClick={()=>setActiveTab('eps')} className={`flex-1 py-4 text-sm font-bold ${activeTab==='eps' ? 'text-white border-b-2 border-[#0a84ff] bg-white/5' : 'text-[#8e8e93]'}`}>Daftar Episode</button>
              <button onClick={()=>setActiveTab('comments')} className={`flex-1 py-4 text-sm font-bold ${activeTab==='comments' ? 'text-white border-b-2 border-[#0a84ff] bg-white/5' : 'text-[#8e8e93]'}`}>Komentar</button>
            </div>

            {/* Content Area */}
            <div className="flex-1 overflow-y-auto no-scrollbar p-4">
              {activeTab === 'eps' ? (
                <>
                  <input type="text" placeholder="Cari episode..." className="w-full bg-black/50 border border-white/10 rounded-lg px-4 py-2 text-sm text-white mb-4 focus:outline-none focus:border-[#0a84ff]" />
                  {/* Anggap EpisodeList menerima props 'currentActive' untuk di-highlight dengan background biru */}
                  <EpisodeList episodes={episodes} animeId={anime.id} currentActive={currentEpisode.number} />
                </>
              ) : (
                <div className="text-center py-20 text-[#8e8e93] text-sm">Fitur Komentar Segera Hadir</div>
              )}
            </div>
          </div>
        </div>

      </div>
      {showShortcuts && <ShortcutModal onClose={() => setShowShortcuts(false)} />}
    </div>
  );
}

⚡ 7. Saran Peningkatan #5: Additional Player Features

Setelah fondasi utama player dibenahi, fitur-fitur berikut akan memberi rasa premium polish. Masing-masing cukup realistis untuk ditambahkan bertahap tanpa mengacak arsitektur yang sudah ada.

1. Picture-in-Picture (PiP) 📺

Manfaat UX: User tetap bisa menonton sambil menjelajahi daftar episode, halaman detail, atau tab lain.

Catatan Implementasi: Gunakan Browser PiP API, siapkan fallback bila browser tidak mendukung.

const togglePiP = async () => {
  if (!videoRef.current) return;

  try {
    if (document.pictureInPictureElement) {
      await document.exitPictureInPicture();
      return;
    }

    if (document.pictureInPictureEnabled) {
      await videoRef.current.requestPictureInPicture();
    }
  } catch (error) {
    console.error('Gagal membuka Picture-in-Picture', error);
  }
};

2. Screenshot Button 📸

Manfaat UX: Pengguna anime sering menyimpan frame favorit untuk dibagikan ke Discord, X, atau Telegram.

Catatan Implementasi: Hanya berlaku untuk video native, bukan iframe fallback.

const captureScreenshot = () => {
  if (!videoRef.current) return;

  const canvas = document.createElement('canvas');
  canvas.width = videoRef.current.videoWidth;
  canvas.height = videoRef.current.videoHeight;

  const ctx = canvas.getContext('2d');
  if (!ctx) return;

  ctx.drawImage(videoRef.current, 0, 0, canvas.width, canvas.height);

  const link = document.createElement('a');
  link.href = canvas.toDataURL('image/png');
  link.download = `anime-frame-${Date.now()}.png`;
  link.click();
};

3. Playback Resume Toast ⏯️

Manfaat UX: Saat user kembali ke episode yang pernah ditonton, sistem menawarkan lanjut dari timestamp terakhir.

Catatan Implementasi: Gunakan data dari useWatchHistory(), tampilkan hanya jika progres > 30 detik.

useEffect(() => {
  const saved = getProgress(animeSlug, episodeNum);
  if (!saved || saved.timestampSec < 30) return;

  toast(`Lanjutkan dari ${fmt(saved.timestampSec)}?`, 'info', 5000);

4. Long-Press Seek / Fast Preview ⚡

Manfaat UX: Menahan tombol kanan/kiri memberi rasa navigasi cepat ala platform video modern.

Catatan Implementasi: Gunakan keydown dengan properti event.repeat, lalu reset saat keyup.

useEffect(() => {
  const onKeyDown = (e: KeyboardEvent) => {
    if (!videoRef.current) return;

    if (e.code === 'ArrowRight' && e.repeat) {
      videoRef.current.playbackRate = 2;
    }

    if (e.code === 'ArrowLeft' && e.repeat) {
      videoRef.current.currentTime = Math.max(0, videoRef.current.currentTime - 1);
    }
  };

  const onKeyUp = () => {
    if (videoRef.current) videoRef.current.playbackRate = 1;
  };

  window.addEventListener('keydown', onKeyDown);
  window.addEventListener('keyup', onKeyUp);
  return () => {
    window.removeEventListener('keydown', onKeyDown);
    window.removeEventListener('keyup', onKeyUp);
  };
}, []);

5. Volume Slider 🎚️

const [volume, setVolume] = useState(1);

const handleVolumeChange = (nextVolume: number) => {
  setVolume(nextVolume);
  if (videoRef.current) {
    videoRef.current.volume = nextVolume;
    videoRef.current.muted = nextVolume === 0;

};

🏗️ 8. Architecture Recommendations

Player saat ini sudah membuktikan fondasi teknisnya kuat, tetapi file VideoPlayer.tsx mulai menampung terlalu banyak tanggung jawab. Jika tidak direstrukturisasi sekarang, penambahan fitur baru akan menambah risiko regressions.

8.1 Refactor Menjadi Subcomponents

  • PlayerControls: Play/pause, volume, fullscreen, PiP, settings, navigation actions.
  • ProgressBar: Seek bar, buffered bar, hover preview, chapter markers, skip markers.
  • SettingsMenu: Nested menu untuk quality, speed, subtitle, playback options.
  • GestureHandler: Double-tap seek, swipe volume, swipe brightness, ripple feedback.
  • AutoNextOverlay: Countdown akhir episode dan CTA ke episode berikutnya.

8.2 Shared Player State

Gunakan PlayerContext atau usePlayerStore (Zustand) agar state seperti playing, duration, progress, buffered, volume, showControls, showAutoNext, dan activeProvider tidak tersebar di banyak komponen. 

type PlayerState = {
  playing: boolean;
  duration: number;
  progress: number;
  buffered: number;
  volume: number;
  playbackRate: number;
  showControls: boolean;
  activeProvider: string;
  showAutoNext: boolean;
  setProgress: (value: number) => void;
  setVolume: (value: number) => void;
  setShowAutoNext: (value: boolean) => void;
};

8.3 Event-Driven Player Architecture

Dengan event-driven architecture, setiap fitur mendengar event spesifik, bukan saling menumpuk di satu useEffect raksasa.

  • onEnded -> triggerAutoNext()
  • onTimeUpdate -> evaluateSkipIntro()
  • onSourceChange -> preserveCurrentTime()
  • onProviderChange -> warmupPreferredSource()
  • onVisibilityChange -> enableMiniPlayer()

Kenapa Modularisasi Penting?

Karena halaman streaming akan terus bertambah kompleks. Tanpa modularisasi, setiap tambahan kecil akan memperbesar coupling, membuat debugging lambat, pengujian sulit, dan onboarding contributor menjadi mahal.
